SONIC Inference on RTX 6000 Pro

End-to-end guide: install Isaac Lab, load GMR-retargeted G1 motions, and run the SONIC RL policy to produce physically-consistent whole-body robot motion.

Prerequisites

OS
Ubuntu 22.04 / 20.04
GPU
RTX 6000 Pro
Driver
NVIDIA ≥ 535
CUDA
12.x
Disk
≥ 60 GB free
RAM
≥ 32 GB

1 Install NVIDIA Driver + CUDA

Verify the driver is working. If nvidia-smi fails, install the driver first then reboot.

bash
# Verify driver
nvidia-smi   # should show CUDA 12.x and RTX 6000 Pro

# If missing, install:
sudo apt-get install -y nvidia-driver-535
sudo reboot

# After reboot, verify again:
nvidia-smi
⚠ Do not proceed past this step until nvidia-smi shows your GPU. Isaac Sim will not launch without a working GPU driver.

2 Install Miniconda

Skip this step if Miniconda or Anaconda is already installed.

bash
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh -b -p $HOME/miniconda3
source $HOME/miniconda3/bin/activate
conda init bash && source ~/.bashrc

3 Create the Isaac Lab Environment

Isaac Lab requires exactly Python 3.10. Create a dedicated conda env and install PyTorch with CUDA 12.1 support.

bash
# Create env with Python 3.10
conda create -n isaaclab python=3.10 -y
conda activate isaaclab

# Install PyTorch (CUDA 12.1 — compatible with Isaac Sim 4.5)
pip install torch==2.4.0 torchvision==0.19.0 \
    --index-url https://download.pytorch.org/whl/cu121

4 Install Isaac Sim 4.5.0

Isaac Sim 4.5.0 is installed via pip from NVIDIA's private PyPI index. It pairs with Isaac Lab 2.3.2.

bash
pip install isaacsim==4.5.0 \
    isaacsim-rl==4.5.0 \
    isaacsim-replicator==4.5.0 \
    isaacsim-extscache-physics==4.5.0 \
    isaacsim-extscache-kit==4.5.0 \
    isaacsim-extscache-kit-sdk==4.5.0 \
    --extra-index-url https://pypi.nvidia.com

After installing, run a headless smoke test:

bash
isaacsim --headless --help
ℹ The first run caches kit extensions and may take 5–10 minutes without visible output. This is normal — it only happens once.

5 Install Isaac Lab 2.3.2

Clone Isaac Lab at the exact tag this repo was built against and install all sub-packages.

bash
# Clone at the pinned release tag
git clone https://github.com/isaac-sim/IsaacLab.git \
    --branch v2.3.2 ~/IsaacLab
cd ~/IsaacLab

# Install all sub-packages into the isaaclab env
python -m pip install -e source/isaaclab
python -m pip install -e source/isaaclab_assets
python -m pip install -e source/isaaclab_mirage
python -m pip install -e source/isaaclab_rl
python -m pip install -e source/isaaclab_tasks

# Verify
python -c "import isaaclab; print('IsaacLab OK')"

6 Clone the GR00T-WholeBodyControl Repo

bash
cd ~
git clone https://github.com/nvlabs/GR00T-WholeBodyControl.git
cd GR00T-WholeBodyControl

7 Install gear_sonic[training]

With the isaaclab env still active, install the SONIC training stack.

bash
cd ~/GR00T-WholeBodyControl

# Installs hydra-core, wandb, trl, transformers, accelerate, etc.
pip install -e "gear_sonic[training]"

# Final import check
python -c "import gear_sonic; import isaaclab; print('All imports OK')"

8 Transfer Data Files

Two files need to land on the GPU machine. Choose one of the two approaches below.

Option A — Transfer from the source machine (faster)

Run these commands from the source machine. Replace <USER> and <GPU_MACHINE_IP> with your details.

bash
DEST="<USER>@<GPU_MACHINE_IP>:~/GR00T-WholeBodyControl"

# SONIC checkpoint (~448 MB)
rsync -avz --progress sonic_release/ $DEST/sonic_release/

# Converted motion library (~3.4 MB)
rsync -avz --progress \
    gear_sonic/data/gmr_test10_motion_lib.pkl \
    $DEST/gear_sonic/data/gmr_test10_motion_lib.pkl

Option B — Regenerate on the GPU machine

If you prefer not to transfer, download the checkpoint and re-run the converter on the GPU machine.

bash
# On the GPU machine — download checkpoint from HuggingFace
conda activate isaaclab
cd ~/GR00T-WholeBodyControl
python download_from_hf.py --training --no-smpl

# Transfer the raw GMR PKLs from the source machine first:
rsync -avz <USER>@<SOURCE_IP>:/home/ubuntu/menga/GMR/outputs/test_10/pkls/ \
    ~/gmr_pkls/

# Re-run the converter
python gear_sonic/data_process/convert_gmr_to_motion_lib.py \
    --input ~/gmr_pkls \
    --output gear_sonic/data/gmr_test10_motion_lib.pkl
📁
After this step you should have:
~/GR00T-WholeBodyControl/sonic_release/last.pt (448 MB)
~/GR00T-WholeBodyControl/sonic_release/config.yaml
~/GR00T-WholeBodyControl/gear_sonic/data/gmr_test10_motion_lib.pkl (3.4 MB)

9 Run SONIC Inference

All commands run from ~/GR00T-WholeBodyControl with the isaaclab env active.

⚠ Always run from the repo root. Hydra changes the working directory internally but the script calls os.chdir(hydra.utils.get_original_cwd()) to restore it — this only works correctly if you start from the repo root.

9a. Metrics — headless, no display required

bash
conda activate isaaclab
cd ~/GR00T-WholeBodyControl

python gear_sonic/eval_agent_trl.py \
    +checkpoint=sonic_release/last.pt \
    +headless=True \
    ++eval_callbacks=im_eval \
    ++run_eval_loop=False \
    ++num_envs=10 \
    "+manager_env/terminations=tracking/eval" \
    "++manager_env.commands.motion.motion_lib_cfg.motion_file=gear_sonic/data/gmr_test10_motion_lib.pkl" \
    "++manager_env.commands.motion.motion_lib_cfg.smpl_motion_file=dummy"

9b. Render Videos — saves MP4s to /tmp/sonic_renders/

bash
python gear_sonic/eval_agent_trl.py \
    +checkpoint=sonic_release/last.pt \
    +headless=True \
    ++eval_callbacks=im_eval \
    ++run_eval_loop=False \
    ++num_envs=10 \
    ++manager_env.config.render_results=True \
    "++manager_env.config.save_rendering_dir=/tmp/sonic_renders" \
    ++manager_env.config.env_spacing=10.0 \
    "~manager_env/recorders=empty" "+manager_env/recorders=render" \
    "++manager_env.commands.motion.motion_lib_cfg.motion_file=gear_sonic/data/gmr_test10_motion_lib.pkl" \
    "++manager_env.commands.motion.motion_lib_cfg.smpl_motion_file=dummy"

Troubleshooting

isaacsim hangs on first run

Isaac Sim caches its kit extensions on the first launch. This is normal and takes 5–10 minutes with no visible progress. Leave the process running. It will complete and subsequent launches will be fast.

ImportError: isaaclab

The isaaclab conda env is not active, or Isaac Lab was installed into a different env. Run conda activate isaaclab before calling eval_agent_trl.py.

Verify with: python -c "import isaaclab; print(isaaclab.__version__)"

CUDA out of memory

num_envs=10 is conservative and should fit comfortably in 48 GB VRAM. If you still see OOM errors, drop the env count:

bash
++num_envs=4
motion_file path errors

Always run the eval command from the repo root (~/GR00T-WholeBodyControl). The script restores the original CWD via Hydra, so relative paths are resolved from there. Using an absolute path also works:

bash
"++manager_env.commands.motion.motion_lib_cfg.motion_file=$HOME/GR00T-WholeBodyControl/gear_sonic/data/gmr_test10_motion_lib.pkl"
smpl_motion_file errors

The literal string dummy is the correct value — it signals motion_lib to skip SMPL data and operate in robot-only tracking mode. Do not replace it with a real path unless you have the SMPL dataset downloaded.

Pip install fails — isaacsim not found

The NVIDIA PyPI index must be included explicitly:

bash
pip install isaacsim==4.5.0 --extra-index-url https://pypi.nvidia.com

If that index is unreachable from your network, you may need to configure a proxy or download the wheels manually from the NVIDIA developer portal.
